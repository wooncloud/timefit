# 타임핏(TimeFit) 고객용 모바일 웹 서비스 기획 (v1.0)

## 1. 개요
타임핏 서비스의 엔드 유저(일반 고객)를 위한 모바일 최적화 웹 페이지 기획입니다. 사업자 위주의 기존 시스템에서 벗어나, 일반 사용자가 업체를 탐색하고 손쉽게 예약할 수 있는 환경을 제공하는 것을 목표로 합니다.

## 2. 주요 타겟 및 목적
- **타겟**: 뷰티, 스포츠, 카페, 전시장 등 예약 기반 서비스를 이용하려는 일반 소비자.
- **목적**: 
  - 내 주변 혹은 특정 지역의 업체 검색 및 정보 확인.
  - 실시간 예약 및 일정 관리.
  - 내가 이용한 서비스 내역 관리.

## 3. 핵심 사용자 시나리오 (User Flow)
1. **업체 탐색**: 홈 화면 -> 카테고리 선택/검색 -> 업체 리스트 -> 업체 상세 페이지
2. **예약 진행**: 업체 상세 (메뉴 선택) -> 상세 메뉴 정보 -> 예약/스케줄 선택 -> 예약 확정
3. **히스토리 확인**: 마이 페이지 -> 내 예약 내역 -> 예약 상세 확인/취소

## 4. 상세 화면 기획 (Mobile Layout)

### 4.1. 홈 화면 (Home)
- **주요 구성**: 
  - **헤더**: 로고, 알림 아이콘.
  - **검색 바**: 홈 상단에 고정된 검색 필드.
  - **카테고리 그리드**: (예: 음식점, 카페, 스포츠, 뷰티, 건강 등) 아이콘 형태의 퀵 메뉴.
  - **추천/주변 업체**: "내 주변 인기 업체", "새로 등록된 업체" 등 큐레이션 섹션 (카드 레이아웃).
  - **하단 네비게이션 (Fixed)**: 
    - **홈**: 서비스 메인 화면.
    - **내 예약**: 실시간 예약 현황 및 히스토리 관리 (Badge 표시).
    - **찜**: 즐겨찾기한 업체 리스트.
    - **마이**: 프로필, 리뷰 관리, 환경 설정.

### 4.2. 검색 및 필터 페이지 (Search & List)
- **주요 구성**:
  - **검색 바**: 검색어 입력 필드.
  - **필터링**: 정렬(인기순, 최신순), 업종 필터 (MVP에서는 텍스트/카테고리 기반 검색에 집중).
  - **업체 리스트**: 
    - 썸네일 이미지 (4:3 또는 1:1 비율).
    - 업체명, 업종, 간단한 설명(Description).
    - 주소(텍스트), 평점(추후 확장), 예약 가능 여부 태그.

### 4.3. 업체 상세 페이지 (Business Detail)
- **주요 구성**:
  - **상단 비주얼**: 업체 대표 이미지 슬라이드 (Cover Image).
  - **기본 정보**: 업체명, 로고, 즐겨찾기(북마크) 버튼.
  - **탭 메뉴 구성**:
    - **정보**: 업체 소개(Description), 공지사항(Notice), 전화번호, 영업시간 (지도 제외).
    - **서비스/상품**: 제공 중인 메뉴/서비스 리스트 (이미지 + 명칭 + 가격 + 소요시간).
    - **리뷰**: 사용자들이 남긴 후기 (추후 확장).
  - **플로팅 예약 버튼**: 하단에 고정되어 서비스/메뉴 선택으로 바로 이동.

### 4.4. 서비스/상품 상세 (Service Detail)
- **주요 구성**:
  - 서비스 이미지, 명칭, 가격.
  - 상세 설명 (서비스 과정, 소요시간 등).
  - **예약하기 버튼**: 클릭 시 날짜/시간 선택 화면으로 연결.

### 4.5. 예약 일정 선택 (Booking Slot Selection)
- **주요 구성**:
  - **캘린더**: 월별/주별 날짜 선택 (가로 스크롤 혹은 전체 캘린더).
  - **시간 슬롯**: 선택한 날짜의 예약 가능 시간대 (버튼 그리드 형태).
  - **인원 선택**: (필요 시) 예약 인원 설정.

### 4.6. 예약 확인 및 결제/확정 (Booking Confirmation)
- **주요 구성**:
  - 예약 정보 요약 (업체명, 서비스, 일시, 인원).
  - 예약자 정보 입력/확인 (성함, 연락처, 요청사항).
  - 동의 사항 (취소 규정 등).
  - **최종 예약 완료 버튼**.

### 4.7. 마이페이지 (My Page)
- **주요 구성**:
  - **회원 프로필**: 닉네임, 프로필 사진, 계정 정보 수정.
  - **활동 요약**: 작성한 리뷰 개수, 찜한 업체 수 등 요약 정보.
  - **설정 및 기타**: 알림 설정, 공지사항, 고객센터, 약관 및 정책, 로그아웃.

### 4.8. 내 예약 (Bookings) - [독립 탭]
- **주요 구성**:
  - **상태별 탭**: '이용 예정(입금대기/확정)', '이용 완료', '취소/노쇼'로 내역 구분.
  - **예약 카드 UI**:
    - 업체명 및 서비스명.
    - 예약 일시 (예: 2024.01.20 14:00).
    - 상태 태그 (예: 예약확정, 방문완료).
    - 액션 버튼: '예약 상세 보기', '리뷰 쓰기'(이용 완료 시), '예약 취소'(이용 예정 시).
  - **엠프티 케이스(Empty Case)**: 예약 내역이 없을 때의 안내 문구 및 '업체 보러 가기' 버튼.

### 4.9. 찜 / 단골 (Favorites) - [독립 탭]
- **주요 구성**:
  - **업체 리스트**: 즐겨찾기(하트)를 누른 업체들만 모아서 노출.
  - **리스트 정보**: 업체 썸네일, 업체명, 주소, 별점/리뷰 수.
  - **빠른 액션**: 각 카드 내 '예약하기' 버튼을 배치하여 반복 방문 유도.
  - **엠프티 케이스**: 찜한 업체가 없을 때 "나만의 단골 업체를 추가해 보세요!" 안내.

## 5. 디자인 가이드라인 (Designer's Tip)
- **Mobile-First**: 모든 레이아웃은 모바일(360px ~ 430px)을 기준으로 최적화합니다.
- **Bottom Navigation**: 엄지손가락 범위 내에서 주요 기능을 이동할 수 있도록 하단 탭바를 적극 활용합니다.
- **Micro-interactions**: 버튼 탭 효과, 페이지 전환 애니메이션, 로딩 스켈레톤 등을 통해 고급스러운 사용자 경험을 제공합니다.
- **Visual Style**: 
  - 신뢰감을 주는 모던한 서체(Suit, Pretendard 등) 사용.
  - 선명한 고품질 이미지를 위한 배치.
  - 타임핏 브랜드 컬러 포인트를 적절히 배치.

## 6. 데이터 연동 필요 항목
- `PublicBusinessDetail`: 업체 기본 정보.
- `Menu`: 예약 가능한 서비스/상품 리스트.
- `OperatingHours`: 영업시간 및 휴무일 정보.
- `BookingSlots`: 실시간 예약 가능 시간대.

## 7. 권장 파일 구조 (Implementation Path)
기존 프로젝트 구조와 일관성을 유지하기 위해 `(customer)` 그룹 폴더 생성을 권장합니다.

- `src/app/(customer)/`
  - `page.tsx`: 고객용 홈 (탐색)
  - `search/page.tsx`: 검색 결과 및 필터 리스트
  - `b/[businessId]/page.tsx`: 업체 상세 (정보/메뉴/리뷰)
  - `b/[businessId]/menu/[menuId]/page.tsx`: 서비스/상품 상세
  - `b/[businessId]/book/page.tsx`: 예약 일정 선택 및 확정
  - `mypage/`: 마이페이지 관련 (예약 내역, 프로필)
